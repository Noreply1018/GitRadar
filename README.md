# GitRadar

GitRadar 是一个面向个人使用的开源项目发现工具。

目标是每天自动抓取 GitHub 上值得关注的开源项目，并以一种低维护、低噪音的方式呈现出来，帮助我快速判断今天有哪些项目值得看、值得收藏、值得后续跟进。

## 当前定位

- 这是一个自用项目，不以多用户产品为前提。
- 第一优先级是实用和维护成本，而不是功能堆叠。
- 输出形式暂未确定，会结合开发难度、使用成本和实际体验再定。

## 第一批要回答的问题

1. 数据源只看 GitHub，还是要混合其他来源。
2. “优秀项目”的判断标准是什么。
3. 每天希望看到多少个项目。
4. 输出形式是网页、命令行、邮件、Telegram，还是别的形式。
5. 需要多强的历史归档、收藏和检索能力。

## 初步范围

第一版优先考虑：

- 每日定时抓取
- 基础筛选和排序
- 生成可读的项目列表
- 支持手动回看历史结果

暂不默认纳入：

- 多用户系统
- 社交互动
- 复杂推荐算法
- 过度依赖外部付费服务

## 目录规划

当前仓库刚初始化，目录结构会在需求收敛后再补齐，避免过早设计。
