# GitRadar

GitRadar 是一个面向个人使用的开源项目发现工具。

目标是每天自动抓取 GitHub 上值得关注的开源项目，并以一种低维护、低噪音的方式呈现出来，帮助我快速判断今天有哪些项目值得看、值得收藏、值得后续跟进。

## 当前定位

- 这是一个自用项目，不以多用户产品为前提。
- 第一优先级是实用和维护成本，而不是功能堆叠。
- 输出形式优先考虑手机端易消费的形态。

## 当前共识

- 项目名：`GitRadar`
- 使用对象：只给自己用
- 项目范围：全部都看，但偏向有意思、前沿、值得关注的项目
- 策略偏好：宁可少，也要准
- 自动化程度：全自动
- 输出优先级：`Telegram 推送` 优先
- 每日数量：`5 个以内`
- 可接受接入模型 API，用于摘要、解释和筛选增强

GitRadar v1 现在更像一个“每日精选开源项目雷达”：

- 每天自动抓取候选项目
- 自动筛选出少量真正值得看的项目
- 通过手机上方便查看的方式推送给我
- 每个项目除了链接，还要解释它为什么值得看

## 每条项目最好回答的问题

- 它是干什么的
- 为什么值得看
- 和别的项目相比新在哪
- 我适不适合点进去深看
- 最近热度是否明显上升

## 下一步待确认

1. 候选数据源只看 GitHub，还是补充 Hacker News / Reddit 等外部来源。
2. “前沿”和“有意思”的筛选规则如何落地。
3. 推送时间是固定每天一次，还是支持手动触发补抓。
4. 是否需要一个简单网页做历史归档。
5. 模型调用是在抓取阶段生成摘要，还是只对入选项目生成摘要。

## 初步范围

第一版优先考虑：

- 每日定时抓取
- 基础筛选和排序
- 生成少量高质量项目卡片
- Telegram 推送
- 支持后续补一个简单归档入口

暂不默认纳入：

- 多用户系统
- 社交互动
- 复杂推荐算法
- 过度依赖外部付费服务
- 一开始就做大而全的数据平台
